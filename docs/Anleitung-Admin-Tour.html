Anleitung: Admin-Tour in einem WordPress-Plugin

Nach dem Vorbild des Plugins WP AI – für die Umsetzung in anderen Plugins.

Überblick

WP AI bietet zwei Tour-Typen:

TourZweckWordPress-Komponente
Guided TourEinführung mit mehreren Info-Seiten (Modal)@wordpress/components → Guide
Setup TourKontextuelle Schritt-für-Schritt-Anleitung auf der EinstellungsseiteEigenes React-Overlay + data-*-Anker

1. Dateistruktur (wp-ai)

src/js/
  wp-ai-guided-tour.js   ← Einstiegspunkt, rendert React-App
  setup-tour.js                 ← Kontextuelle Setup-Tour
  setup-tour-step.js            ← Wiederverwendbares Schritt-Panel

includes/Common/Settings/
  Settings.php                  ← Script enqueue, AJAX, User-Meta
  templates/settings-page.php   ← Buttons + React-Root
  templates/tab-menu.php        ← data-bk-tour auf Tabs
  templates/options/*.php       ← data-bk-tour auf Formularfeldern

src/sass/wp-ai-admin.scss
build/wp-ai-guided-tour.js

2. PHP: Script laden und Konfiguration übergeben

In Settings.php werden Hooks registriert, das Script nur auf der Plugin-Seite geladen und per wp_localize_script Daten an JavaScript übergeben.

Für dein Plugin anpassen:

  1. admin_enqueue_scripts – nur auf der relevanten Admin-Seite
  2. wp_enqueue_script mit .asset.php aus dem Build
  3. wp_enqueue_style('wp-components') und dashicons
  4. wp_set_script_translations() für Übersetzungen
  5. wp_localize_script() mit autoStart, activeTab, settingsUrl, ajaxUrl, Nonces
  6. Zwei AJAX-Handler zum Speichern in User Meta (*_dismissed)
add_action('admin_enqueue_scripts', [$this, 'enqueueGuidedTour']);
add_action('wp_ajax_wp_ai_dismiss_guided_tour', [$this, 'dismissGuidedTour']);
add_action('wp_ajax_wp_ai_dismiss_setup_tour', [$this, 'dismissSetupTour']);

wp_localize_script('wp-ai-guided-tour', 'BKWPAIGuide', [
    'autoStart'      => !get_user_meta(..., 'wp_ai_guided_tour_dismissed', true),
    'autoStartSetup' => isset($_GET['bk_setup_tour']),
    'setupTourStepId'=> $setupTourStepId,
    'settingsUrl'    => $this->getUrl(),
    'activeTab'      => $this->getActiveTab()->slug,
    'ajaxUrl'        => admin_url('admin-ajax.php'),
    'nonce'          => wp_create_nonce('wp_ai_guided_tour'),
]);

3. HTML: Root-Container und Start-Buttons

<button type="button" id="wp-ai-start-guided-tour" class="page-title-action">
    Guided tour
</button>
<button type="button" id="wp-ai-start-setup-tour" class="page-title-action">
    Setup tour
</button>
<div id="wp-ai-guided-tour-root"></div>

React mountet in #wp-ai-guided-tour-root. Die Button-IDs müssen mit dem JavaScript übereinstimmen.

4. Anker im Markup: data-bk-tour

Die Setup-Tour findet UI-Elemente per CSS-Selektor. Setze ein einheitliches Attribut:

  • Tabs: data-bk-tour="tab-domains"
  • Formularfelder: data-bk-tour="new-domain" (bedingt im Template)
  • Speichern-Button: data-bk-tour="save-settings"

Namenskonvention: Für ein anderes Plugin z. B. data-meinplugin-tour="eindeutige-id".

5. Guided Tour (einfache Einführung)

Die Guided Tour nutzt die WordPress-Komponente Guide:

import { Guide } from '@wordpress/components';

<Guide
    className="wp-ai-guided-tour"
    finishButtonText={__('Get started', 'wp-ai')}
    onFinish={finishGuide}
    pages={guidePages}
/>

Schritte:

  1. Guide aus @wordpress/components importieren
  2. guidePages als Array mit image und content definieren
  3. Beim Beenden per AJAX dismissed in User Meta speichern
  4. autoStart nur, wenn der User die Tour noch nicht abgeschlossen hat

6. Setup Tour (kontextuelle Schritt-für-Schritt-Tour)

6.1 Schritte definieren

{
    id: 'new-domain',
    tab: 'domains',
    target: '[data-bk-tour="new-domain"]',
    title: __('Add a domain', 'wp-ai'),
    text: __('Enter the URL...', 'wp-ai'),
    optional: true,
}
FeldBedeutung
idEindeutige Schritt-ID (für URL-Parameter)
tabTab-Slug, auf dem der Schritt sichtbar ist
targetCSS-Selektor zum DOM-Element
title / textAnzeige im Dialog
optionalSchritt wird übersprungen, wenn target fehlt

6.2 Tab-Wechsel über URL

Wechselt ein Schritt den Tab, lädt die Seite neu mit:

?tab=import&bk_setup_tour=1&bk_setup_tour_step=import-categories

PHP liest bk_setup_tour_step und übergibt setupTourStepId an JavaScript.

6.3 Highlight und Scroll

Das Ziel-Element erhält die CSS-Klasse wp-ai-setup-tour__highlight und wird per scrollIntoView in den sichtbaren Bereich gescrollt.

6.4 Schritt-Panel

setup-tour-step.js nutzt WordPress-Button mit Zurück, Überspringen und Weiter/Fertig.

7. CSS

  • __overlay – halbtransparenter Hintergrund (z-index: 99990)
  • __highlight – blauer Outline um das Ziel-Element (z-index: 99995)
  • __card – fixiertes Panel unten mittig (z-index: 100000)

Styles in src/sass/wp-ai-admin.scss, Build mit sass nach build/css/.

8. Build-Pipeline

"build:js": "wp-scripts build ... src/js/wp-ai-guided-tour.js"
npm run build:js
npm run build:css

Ergebnis: build/wp-ai-guided-tour.js + .asset.php

9. Checkliste für ein neues Plugin

Minimal (nur Guided Tour)

  • src/js/mein-plugin-guided-tour.js mit Guide
  • React-Root + Start-Button im Admin-Template
  • enqueueGuidedTour() in PHP
  • AJAX-Handler dismiss_*_tour
  • User Meta mein_plugin_guided_tour_dismissed
  • SCSS + Build
  • Strings mit __() und Textdomain

Mit Setup Tour zusätzlich

  • data-meinplugin-tour an Tabs, Feldern, Buttons
  • setup-tour.js mit Schritt-Array
  • URL-Parameter für Tab-Wechsel
  • activeTab per PHP an JS übergeben
  • buildTourPath() – nur Schritte mit sichtbarem target
  • Overlay + Highlight-CSS

11. Zusammenfassung

WP AI nutzt WordPress-eigene Bausteine (Guide, Button, @wordpress/element) statt einer externen Tour-Library. Die Guided Tour erklärt das Plugin allgemein; die Setup Tour verknüpft Schritte mit echten UI-Elementen über data-bk-tour-Anker und Tab-Navigation per URL.

Für ein anderes Plugin: Präfixe, Schritte und Anker anpassen, Build-Pipeline einrichten – die Architektur bleibt gleich.
